# Data

Local IEMOCAP files live here. The raw dataset and generated metadata are ignored by git.

Expected organization:

```text
data/
  README.md
  metadata.csv                  # generated by scripts/build_iemocap_metadata.py
  iemocap_dataset_analysis.ipynb
  Session1/
    dialog/
      EmoEvaluation/
        Ses01F_impro01.txt
        ...
      transcriptions/
        Ses01F_impro01.txt
        ...
    sentences/
      wav/
        Ses01F_impro01/
          Ses01F_impro01_F000.wav
          Ses01F_impro01_F001.wav
          ...
  Session2/
    ...
  Session3/
    ...
  Session4/
    ...
  Session5/
    ...
```

The project config points `iemocap_root` to `data`, so scripts expect the `Session1` through `Session5` folders directly inside this directory.

Generate metadata with:

```powershell
.\.venv\Scripts\python.exe scripts\build_iemocap_metadata.py
```

Generated `metadata.csv` columns:

```csv
audio_id,audio_path,true_label,session,dialog_id,start,end,valence,activation,dominance
```

The metadata builder keeps the main emotion labels used by the project:

```text
ang, exc, hap, neu, sad
```

For the SUPERB 4-class IEMOCAP model used by Pastor et al., evaluation code maps `exc` to `hap`.
